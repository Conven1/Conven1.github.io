Project analysis

发表于2025-03-19|Curriculum vitae

项目经历

xx社区分享平台

技术栈：Spring Cloud/Spring Boot/MySQL/Redis/Vue.js+ElementUI

角色：全栈开发

项目描述：一款UGC内容社区平台，支持文章/视频分享、个性化推荐、互动社交及积分激励体系。

  1. 核心功能模块
  • 内容发布
    • 设计混合存储架构：文章、评论等用MySQL分表存储，提高查询效率；图片/视频用OSS+CDN加速，降低上传耗时
    • 构建内容审核体系：文本敏感词过滤(AC自动机+正则匹配)、阿里云内容安全API识别多媒体违规内容、人工审核后台拦截违规内容
  • 个性化推荐方案
    • 实现混合推荐策略：协同过滤(User/Item CF)+热度加权(PV/UV衰减模型)，提升内容互动率
    • 多语言解决方案：集成Google Translate API实现评论/文章实时翻译，缓存翻译结果(Redis TTL 24h)降低API调用成本
  • 社交互动
    • 点赞/收藏计数：Redis INCR原子操作 + 定时落库，QPS 5000+时响应时间<50ms
    • 优化评论树形结构查询：通过MPTT算法改造评论树结构，提升嵌套查询性能
    • 实时通信：基于WebSocket长连接实现实时私信，结合Redis的Pub/Sub解决多实例消息同步问题
  1. 附加功能模块
  • 积分激励机制
    • 设计最终一致性方案：积分流水表(MySQL)与Redis缓存组合方案，保证并发积分兑换业务零差错
    • 任务系统驱动用户行为：开发签到/课程学习/积分抽奖等积分激励系统，提升用户留存率
  • 高可用消息通知
    • 异步通知队列：基于Redisson分布式队列，解耦通知发送避免主业务阻塞，主业务延迟无感知
    • 通知防刷方案：通过Redis过期键实现短信/邮件的防重发控制，结合阿里云短信API(重试+熔断)，降低短信成本

功能解析

设计混合存储架构

文章用MySQL分表(按用户ID哈希)，图片/视频用OSS+CDN加速，降低上传耗时

具体实现方式

  1. MySQL分表存储
    按用户ID哈希分表,分成article_0、article_1、…、article_15共16张表。存储时，计算user_id % 16决定存入哪张表。
  2. 图片/视频用OSS+CDN加速
    前端上传文件,后端生成唯一文件名,调用阿里云OSS SDK，直接上传到OSS，返回访问URL,存储OSS URL到MySQL。
    用户访问图片/视频时，优先走CDN缓存。CDN 未命中时，回源到OSS拉取，并缓存到CDN节点。

相关问题

  1. 分表后，如何做全量搜索？
    Elasticsearch方案或阿里云OpenSearch方案，采用的是阿里云OpenSearch方案
